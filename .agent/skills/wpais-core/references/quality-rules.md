# Quality Rules

## 백엔드

- 리소스 중심 REST 경로를 유지합니다.
- DTO 검증을 생략하지 않습니다.
- Repository 메서드를 우선하고 raw query는 최소화합니다.
- 빈 `catch`를 두지 않습니다.
- `console.*`보다 logger를 우선합니다.

## 보안

- 시크릿과 자격 증명 하드코딩 금지
- 토큰, 비밀번호, 개인정보 로그 금지
- 입력 검증, 권한 체크, 파라미터 바인딩 확인
- 민감 파일 편집 시 커밋 대상 재확인

## 테스트

- 관련 테스트를 우선 실행합니다.
- 새 로직에는 최소 단위 테스트를 권장합니다.
- 중요한 플로우는 통합 테스트 또는 E2E 관점으로 확인합니다.
- 테스트명은 기대 동작과 조건이 드러나게 작성합니다.

## 리뷰

- 기능 정확성
- 회귀 위험
- 보안 영향
- 테스트 누락
- API/문서 계약 불일치
