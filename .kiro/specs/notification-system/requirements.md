# Requirements: 알림 시스템 (Notification System)

## Overview

WPAIS 웹툰 제작 관리 시스템의 알림 기능을 확장합니다. 기존 Alert 시스템을 기반으로 마감 임박 알림, 작업자 알림, 알림 구독/설정 기능을 추가합니다.

## User Stories

### 1. 마감 임박 알림

#### 1.1 공정별 마감 임박 알림
**As a** 프로젝트 관리자
**I want to** 각 공정(배경, 선화, 채색, 후보정)의 마감이 임박했을 때 알림을 받고 싶다
**So that** 마감 전에 작업 진행 상황을 확인하고 대응할 수 있다

**Acceptance Criteria:**
- 공정 마감 3일 전, 1일 전, 당일에 알림 발생
- 알림에 에피소드 번호, 페이지 번호, 공정 유형, 남은 시간 포함
- 이미 완료된 공정은 알림 발생하지 않음

#### 1.2 에피소드 마감 임박 알림
**As a** 프로젝트 관리자
**I want to** 에피소드 전체 마감이 임박했을 때 알림을 받고 싶다
**So that** 에피소드 완료를 위한 최종 점검을 할 수 있다

**Acceptance Criteria:**
- 에피소드 마감 7일 전, 3일 전, 1일 전에 알림 발생
- 알림에 에피소드 번호, 완료율, 남은 작업 수 포함
- 이미 완료된 에피소드는 알림 발생하지 않음

#### 1.3 마일스톤 마감 임박 알림
**As a** 프로젝트 관리자
**I want to** 마일스톤 마감이 임박했을 때 알림을 받고 싶다
**So that** 중요 일정을 놓치지 않을 수 있다

**Acceptance Criteria:**
- 마일스톤 마감 14일 전, 7일 전, 3일 전, 1일 전에 알림 발생
- 알림에 마일스톤 이름, 유형, 남은 일수 포함
- 이미 완료된 마일스톤은 알림 발생하지 않음

### 2. 작업 상태 변경 알림

#### 2.1 작업 시작 알림
**As a** PD(프로젝트 관리자)
**I want to** 작업이 시작되었을 때 알림을 받고 싶다
**So that** 작업 진행 상황을 실시간으로 파악할 수 있다

**Acceptance Criteria:**
- 공정 상태가 READY → IN_PROGRESS로 변경 시 알림 발생
- 알림에 에피소드 번호, 페이지 번호, 공정 유형 포함
- PD에게 무조건 알림 전송

#### 2.2 작업 완료 → 다음 공정 담당자 알림
**As a** 다음 공정 담당 작업자
**I want to** 이전 공정이 완료되었을 때 알림을 받고 싶다
**So that** 내 작업을 바로 시작할 수 있다

**Acceptance Criteria:**
- 공정 완료 시 다음 공정 담당자에게 알림 발송
  - 배경 완료 → 선화 담당자에게 알림
  - 선화 완료 → 채색 담당자에게 알림
  - 채색 완료 → 후보정 담당자에게 알림
- 알림에 에피소드 번호, 페이지 번호, 완료된 공정, 내 공정 마감일 포함
- PD에게도 동시에 알림 전송 (모든 작업 완료 알림은 PD에게 전송)

#### 2.3 작업 완료 알림 (PD용)
**As a** PD(프로젝트 관리자)
**I want to** 모든 작업 완료 알림을 받고 싶다
**So that** 전체 진행 상황을 파악하고 관리할 수 있다

**Acceptance Criteria:**
- 모든 공정 완료 시 PD에게 알림 발송
- 알림에 에피소드 번호, 페이지 번호, 완료된 공정, 다음 공정 정보 포함
- PD는 알림 설정과 관계없이 모든 알림 수신

#### 2.4 에피소드 완료 알림
**As a** PD(프로젝트 관리자)
**I want to** 에피소드가 완료되었을 때 알림을 받고 싶다
**So that** 버퍼 상태와 전체 진행률을 확인할 수 있다

**Acceptance Criteria:**
- 에피소드의 모든 페이지가 완료되면 알림 발생
- 알림에 에피소드 번호, 완료 시간, 버퍼 상태 포함
- PD에게 무조건 알림 전송

### 3. 알림 수신자 관리

#### 3.1 역할 기반 알림 수신
**As a** 시스템 관리자
**I want to** 역할에 따라 알림 수신자를 자동 지정하고 싶다
**So that** 적절한 사람에게 알림이 전달될 수 있다

**Acceptance Criteria:**
- PD(프로젝트 관리자)는 모든 알림 수신
- 작업자는 자신의 공정 관련 알림만 수신
- 공정별 담당자 지정 가능 (배경, 선화, 채색, 후보정)

#### 3.2 공정별 담당자 지정
**As a** PD(프로젝트 관리자)
**I want to** 각 공정의 담당자를 지정하고 싶다
**So that** 작업 완료 시 다음 담당자에게 알림을 보낼 수 있다

**Acceptance Criteria:**
- 프로젝트별로 공정 담당자 지정 가능
- 담당자 미지정 시 PD에게만 알림 전송
- 담당자 변경 시 이후 알림부터 새 담당자에게 전송

### 4. 알림 설정

#### 4.1 알림 유형별 활성화/비활성화
**As a** 사용자
**I want to** 받고 싶은 알림 유형을 선택하고 싶다
**So that** 필요한 알림만 받을 수 있다

**Acceptance Criteria:**
- 알림 유형별로 활성화/비활성화 설정 가능
- 설정은 프로젝트별로 관리
- 기본값은 모든 알림 활성화

#### 4.2 알림 임계값 설정
**As a** 사용자
**I want to** 마감 임박 알림의 임계값을 설정하고 싶다
**So that** 프로젝트 특성에 맞게 알림을 받을 수 있다

**Acceptance Criteria:**
- 공정/에피소드/마일스톤별 알림 임계값(일수) 설정 가능
- 기본값: 공정(3,1,0일), 에피소드(7,3,1일), 마일스톤(14,7,3,1일)

### 5. 알림 조회 및 관리

#### 5.1 알림 목록 조회
**As a** 사용자
**I want to** 프로젝트의 알림 목록을 조회하고 싶다
**So that** 최근 알림을 확인할 수 있다

**Acceptance Criteria:**
- 프로젝트별 알림 목록 조회 가능
- 알림 유형, 심각도, 날짜 범위로 필터링 가능
- 페이지네이션 지원

#### 5.2 알림 확인 처리
**As a** 사용자
**I want to** 알림을 확인 처리하고 싶다
**So that** 확인한 알림과 미확인 알림을 구분할 수 있다

**Acceptance Criteria:**
- 개별 알림 확인 처리 가능
- 여러 알림 일괄 확인 처리 가능
- 확인 처리 시간 기록

#### 5.3 미확인 알림 수 조회
**As a** 사용자
**I want to** 미확인 알림 수를 조회하고 싶다
**So that** 확인이 필요한 알림이 있는지 빠르게 파악할 수 있다

**Acceptance Criteria:**
- 프로젝트별 미확인 알림 수 조회 가능
- 심각도별 미확인 알림 수 조회 가능

### 6. 스케줄러 기반 알림 생성

#### 6.1 정기 마감 체크
**As a** 시스템
**I want to** 정기적으로 마감 임박 항목을 체크하고 싶다
**So that** 마감 임박 알림을 자동으로 생성할 수 있다

**Acceptance Criteria:**
- 매일 정해진 시간에 마감 체크 실행
- 중복 알림 방지 (같은 항목에 대해 같은 임계값 알림은 1회만)
- 체크 실행 로그 기록

## Non-Functional Requirements

### 성능
- 알림 생성 응답 시간 < 100ms
- 알림 목록 조회 응답 시간 < 200ms
- 스케줄러 실행 시간 < 30초 (프로젝트 100개 기준)

### 확장성
- 향후 이메일/슬랙 등 외부 알림 채널 연동 가능한 구조
- 알림 템플릿 시스템 확장 가능

### 데이터 보존
- 알림 데이터 90일 보존
- 90일 이후 자동 삭제 또는 아카이브
