# Requirements: Calendar View

## Overview

웹툰 제작 일정을 캘린더 형태로 시각화하여 PD와 작가가 전체 일정을 한눈에 파악하고 관리할 수 있는 기능을 제공합니다. 에피소드 마감일, 마일스톤, 개별 페이지 작업 일정을 통합하여 직관적인 일정 관리 경험을 제공합니다.

## User Stories

### 1. 캘린더 뷰 기본 기능
As a PD
I want to 프로젝트 일정을 캘린더 형태로 조회
So that 전체 일정을 시각적으로 파악할 수 있습니다

#### Acceptance Criteria
- 1.1 월간 뷰로 한 달 전체 일정 표시
- 1.2 주간 뷰로 일주일 상세 일정 표시
- 1.3 일간 뷰로 하루 타임라인 표시
- 1.4 뷰 타입 간 자유로운 전환
- 1.5 이전/다음 기간으로 네비게이션
- 1.6 오늘 날짜로 빠른 이동

### 2. 에피소드 일정 표시
As a PD
I want to 에피소드 마감일(봉인일)을 캘린더에서 확인
So that 연재 일정을 추적할 수 있습니다

#### Acceptance Criteria
- 2.1 에피소드별 마감일(dueDate) 표시
- 2.2 에피소드 상태별 색상 구분 (PENDING/IN_PROGRESS/COMPLETED)
- 2.3 봉인 여부(isSealed) 시각적 표시
- 2.4 에피소드 클릭 시 상세 정보 팝오버/모달

### 3. 마일스톤 일정 표시
As a PD
I want to 프로젝트 마일스톤을 캘린더에서 확인
So that 주요 일정을 놓치지 않을 수 있습니다

#### Acceptance Criteria
- 3.1 마일스톤 목표일(targetDate) 표시
- 3.2 마일스톤 타입별 아이콘/색상 구분
- 3.3 완료/미완료 마일스톤 시각적 구분
- 3.4 마일스톤 클릭 시 상세 정보 표시

### 4. 페이지 작업 일정 표시
As a 작가
I want to 내 작업 일정을 캘린더에서 확인
So that 작업 계획을 세울 수 있습니다

#### Acceptance Criteria
- 4.1 페이지별 예상 작업 기간 표시
- 4.2 공정별(배경/선화/채색/후보정) 일정 구분
- 4.3 작업 상태별 색상 구분 (LOCKED/READY/IN_PROGRESS/DONE)
- 4.4 작업 클릭 시 해당 페이지로 이동

### 5. 일정 필터링
As a 사용자
I want to 원하는 일정만 필터링하여 조회
So that 필요한 정보에 집중할 수 있습니다

#### Acceptance Criteria
- 5.1 일정 타입별 필터 (에피소드/마일스톤/작업)
- 5.2 에피소드 상태별 필터
- 5.3 공정 타입별 필터
- 5.4 필터 조합 적용
- 5.5 필터 상태 URL 파라미터 반영

### 6. 드래그앤드롭 일정 변경
As a PD
I want to 캘린더에서 드래그앤드롭으로 일정 변경
So that 빠르게 일정을 조정할 수 있습니다

#### Acceptance Criteria
- 6.1 에피소드 마감일 드래그앤드롭 변경
- 6.2 마일스톤 목표일 드래그앤드롭 변경
- 6.3 변경 전 확인 다이얼로그 표시
- 6.4 변경 시 연관 일정 자동 재계산 알림
- 6.5 변경 취소(Undo) 기능

### 7. 일정 상세 보기/수정
As a PD
I want to 캘린더에서 일정 상세 정보 확인 및 수정
So that 상세 페이지 이동 없이 빠르게 관리할 수 있습니다

#### Acceptance Criteria
- 7.1 일정 클릭 시 상세 정보 팝오버 표시
- 7.2 팝오버에서 기본 정보 수정 가능
- 7.3 상세 페이지로 이동 링크 제공
- 7.4 관련 일정 빠른 탐색

### 8. 다중 프로젝트 통합 캘린더
As a PD
I want to 여러 프로젝트 일정을 통합하여 조회
So that 전체 포트폴리오를 관리할 수 있습니다

#### Acceptance Criteria
- 8.1 프로젝트별 색상 구분
- 8.2 프로젝트 선택/해제로 표시 토글
- 8.3 프로젝트별 일정 그룹핑
- 8.4 전체 프로젝트 일정 충돌 표시

### 9. 일정 충돌 및 경고 표시
As a PD
I want to 일정 충돌이나 위험 상황을 캘린더에서 확인
So that 문제를 사전에 인지할 수 있습니다

#### Acceptance Criteria
- 9.1 마감 임박 일정 강조 표시
- 9.2 지연된 일정 경고 표시
- 9.3 일정 충돌 시각적 표시
- 9.4 버퍼 부족 경고 표시

### 10. 반응형 캘린더
As a 사용자
I want to 다양한 디바이스에서 캘린더 사용
So that 어디서든 일정을 확인할 수 있습니다

#### Acceptance Criteria
- 10.1 데스크톱 최적화 (월간 뷰 기본)
- 10.2 태블릿 대응 (주간 뷰 기본)
- 10.3 모바일 대응 (일간 뷰 기본, 스와이프 네비게이션)
- 10.4 터치 제스처 지원

## Technical Notes

- 기존 frontend-dashboard 스펙과 일관된 기술 스택 사용
- 캘린더 라이브러리: FullCalendar 또는 react-big-calendar 검토
- 드래그앤드롭: @dnd-kit 또는 라이브러리 내장 기능 활용
- 백엔드 API: 캘린더 전용 엔드포인트 추가 필요

## Design Direction

- 기존 대시보드 디자인 시스템과 일관성 유지
- 일정 밀도가 높은 날짜 처리 (축약 표시 + 더보기)
- 색상 코딩으로 일정 타입/상태 직관적 구분
- 키보드 네비게이션 지원 (접근성)
