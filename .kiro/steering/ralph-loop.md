# Ralph Loop - 반복 개발 가이드

복잡한 작업을 자동 반복하며 완료까지 진행하는 개발 방법론입니다.

## 개념

Ralph는 **자기 참조 피드백 루프**입니다:
1. 작업 수행
2. 결과 확인 (테스트, 린트 등)
3. 실패 시 수정
4. 완료 조건 충족까지 반복

## 사용 시나리오

### TDD 반복 루프
```
다음 작업을 TDD로 완료해주세요:

1. 실패하는 테스트 작성
2. 최소 구현으로 테스트 통과
3. 테스트 실행
4. 실패 시 디버그 및 수정
5. 필요시 리팩토링
6. 모든 테스트 통과까지 반복
7. 완료 시 <promise>COMPLETE</promise> 출력

작업: [작업 내용]
```

### 기능 구현 루프
```
다음 기능을 반복적으로 구현해주세요:

Phase 1: [첫 번째 단계] (테스트 포함)
Phase 2: [두 번째 단계] (테스트 포함)
Phase 3: [세 번째 단계] (테스트 포함)

각 Phase 완료 후:
- 테스트 실행
- 실패 시 수정
- 통과 시 다음 Phase

모든 Phase 완료 시 <promise>COMPLETE</promise> 출력
```

## 프롬프트 작성 가이드

### ✅ 좋은 예시
```markdown
스케줄링 엔진의 의존성 계산 기능 구현:

완료 조건:
- 모든 단위 테스트 통과
- 순환 의존성 감지 동작
- 크리티컬 패스 계산 정확
- 커버리지 80% 이상

완료 시: <promise>COMPLETE</promise>
```

### ❌ 나쁜 예시
```markdown
의존성 계산 기능 만들어줘.
```

```
6. 모든 테스트 통과 확인

완료 시: <promise>TASK_COMPLETE</promise>
```

```
상태 전이 로직 구현:

4. 동시성 테스트 추가

완료 시: <promise>WORKFLOW_COMPLETE</promise>
```

## 핵심 원칙

### 1. 반복 > 완벽
첫 시도에 완벽할 필요 없음. 루프가 작업을 개선함.

### 2. 실패는 데이터
실패는 예측 가능하고 정보를 제공함. 프롬프트 튜닝에 활용.

### 3. 명확한 완료 조건
자동 검증 가능한 조건 (테스트, 린터) 필수.

### 4. 탈출 조건
무한 루프 방지를 위한 최대 반복 횟수 설정.

## 적합한 작업

**적합:**
- 명확한 성공 기준이 있는 작업
- 반복과 개선이 필요한 작업 (테스트 통과)
- 자동 검증 가능한 작업

**부적합:**
- 인간 판단이 필요한 작업
- 일회성 작업
- 불명확한 성공 기준
