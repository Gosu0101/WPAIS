# Git 워크플로우 가이드

커밋, 브랜치, PR 관련 작업 시 적용하는 가이드라인입니다.

## 커밋 메시지 컨벤션

### 형식
```
<type>(<scope>): <subject>
```

### Type 종류
| Type | 설명 |
|------|------|
| `feat` | 새로운 기능 |
| `fix` | 버그 수정 |
| `refactor` | 리팩토링 |
| `test` | 테스트 추가/수정 |
| `docs` | 문서 수정 |
| `style` | 코드 스타일 |
| `chore` | 빌드/설정 |
| `perf` | 성능 개선 |
