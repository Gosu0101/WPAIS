# 코드 분석 가이드

코드 탐색 및 아키텍처 설계 시 수동으로 적용하는 가이드입니다.

## Code Explorer (코드 탐색)

기존 기능을 분석하고 이해할 때 사용합니다.

### 핵심 미션
특정 기능이 어떻게 작동하는지 진입점부터 데이터 저장까지, 모든 추상화 레이어를 통해 완전히 이해합니다.

### 분석 접근법

#### 1. 기능 발견
- 진입점 찾기 (API, UI 컴포넌트, CLI 명령)
- 핵심 구현 파일 위치 파악
- 기능 경계와 설정 매핑

#### 2. 코드 흐름 추적
- 진입부터 출력까지 호출 체인 따라가기
- 각 단계에서 데이터 변환 추적
- 모든 의존성과 통합 식별
- 상태 변경과 부작용 문서화

#### 3. 아키텍처 분석
- 추상화 레이어 매핑 (프레젠테이션 → 비즈니스 로직 → 데이터)
- 디자인 패턴과 아키텍처 결정 식별
- 컴포넌트 간 인터페이스 문서화
- 횡단 관심사 (인증, 로깅, 캐싱) 파악

#### 4. 구현 세부사항
- 핵심 알고리즘과 데이터 구조
- 에러 핸들링과 엣지 케이스
- 성능 고려사항
- 기술 부채나 개선 영역

### 출력 형식

```markdown
## 기능 분석: [기능명]

### 진입점
- `src/xxx/controller.ts:45` - REST API 엔드포인트
- `src/xxx/resolver.ts:23` - GraphQL 리졸버

### 실행 흐름
1. [단계 1]: 설명 (파일:라인)
2. [단계 2]: 설명 (파일:라인)
...

### 핵심 컴포넌트
| 컴포넌트 | 파일 | 책임 |
|---------|------|------|
| XxxService | src/xxx/xxx.service.ts | 비즈니스 로직 |
| XxxRepository | src/xxx/xxx.repository.ts | 데이터 접근 |

### 아키텍처 인사이트
- 사용된 패턴: [패턴명]
- 레이어 구조: [설명]
- 설계 결정: [설명]

### 의존성
- 내부: [목록]
- 외부: [목록]

### 필수 파일 목록
1. `src/xxx/xxx.ts` - [이유]
2. `src/xxx/xxx.ts` - [이유]
```

## Code Architect (아키텍처 설계)

새 기능의 아키텍처를 설계할 때 사용합니다.

### 핵심 프로세스

#### 1. 코드베이스 패턴 분석
- 기존 패턴, 컨벤션, 아키텍처 결정 추출
- 기술 스택, 모듈 경계, 추상화 레이어 식별
- 유사 기능을 찾아 확립된 접근법 이해

#### 2. 아키텍처 설계
- 발견된 패턴 기반으로 완전한 기능 아키텍처 설계
- 결정적 선택 - 하나의 접근법 선택 후 커밋
- 기존 코드와의 원활한 통합 보장
- 테스트 가능성, 성능, 유지보수성 고려

#### 3. 완전한 구현 청사진
- 생성/수정할 모든 파일 명시
- 컴포넌트 책임, 통합 포인트, 데이터 흐름
- 명확한 단계별 구현 작업 분해

### 출력 형식

```markdown
## 아키텍처 설계: [기능명]

### 발견된 패턴 & 컨벤션
- 패턴 1: [설명] (파일:라인 참조)
- 유사 기능: [기능명] - [접근법]
- 핵심 추상화: [설명]

### 아키텍처 결정
**선택한 접근법**: [접근법명]

**근거**:
- [이유 1]
- [이유 2]

**트레이드오프**:
- 장점: [목록]
- 단점: [목록]

### 컴포넌트 설계
| 컴포넌트 | 파일 경로 | 책임 | 의존성 |
|---------|----------|------|--------|
| XxxController | src/xxx/xxx.controller.ts | API 엔드포인트 | XxxService |
| XxxService | src/xxx/xxx.service.ts | 비즈니스 로직 | XxxRepository |

### 구현 맵
**생성할 파일**:
1. `src/xxx/xxx.controller.ts` - [상세 설명]
2. `src/xxx/xxx.service.ts` - [상세 설명]

**수정할 파일**:
1. `src/xxx/xxx.module.ts` - [변경 내용]

### 데이터 흐름
```
[진입점] → [컴포넌트1] → [컴포넌트2] → [출력]
```

### 구현 순서
- [ ] Phase 1: 기본 구조 설정
- [ ] Phase 2: 핵심 로직 구현
- [ ] Phase 3: 통합 및 테스트
- [ ] Phase 4: 에러 핸들링 및 최적화

### 중요 고려사항
- **에러 핸들링**: [전략]
- **상태 관리**: [접근법]
- **테스팅**: [전략]
- **성능**: [고려사항]
- **보안**: [고려사항]
```
