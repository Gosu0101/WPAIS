# 기능 개발 워크플로우

새로운 기능 개발 시 7단계 프로세스를 따릅니다.

## 7단계 개발 프로세스

```
1. 탐색 → 2. 설계 → 3. 구현 → 4. 리뷰 → 5. 테스트 → 6. 문서 → 7. 배포
```

---

## 1단계: 탐색 (Exploration) 🔍

**목표**: 기존 코드베이스 이해 및 영향 범위 파악

### 체크리스트
- [ ] 관련 기존 코드 파악
- [ ] 의존성 분석
- [ ] 영향받는 모듈 식별
- [ ] 기존 패턴/컨벤션 확인

### 질문
- 비슷한 기능이 이미 있는가?
- 재사용 가능한 코드가 있는가?
- 어떤 모듈에 영향을 주는가?

### 산출물
- 영향 범위 목록
- 재사용 가능 코드 목록
- 기술적 제약사항

---

## 2단계: 설계 (Design) 📐

**목표**: 구현 방향 결정 및 설계 문서화

### 체크리스트
- [ ] 아키텍처 결정
- [ ] 데이터 모델 설계
- [ ] API 인터페이스 정의
- [ ] 에러 처리 전략
- [ ] 테스트 전략

### 산출물
- 설계 문서 (design.md)
- 시퀀스 다이어그램
- 데이터 모델 다이어그램

### WPAIS Spec 연동
```
.kiro/specs/<feature-name>/
├── requirements.md  ← 요구사항
├── design.md        ← 설계
└── tasks.md         ← 구현 계획
```

---

## 3단계: 구현 (Implementation) 💻

**목표**: TDD 방식으로 코드 작성

### 순서
1. 실패하는 테스트 작성
2. 최소 구현
3. 테스트 통과
4. 리팩토링
5. 반복

### 체크리스트
- [ ] 엔티티/모델 구현
- [ ] 서비스 로직 구현
- [ ] 컨트롤러/API 구현
- [ ] 단위 테스트 작성
- [ ] Property-based 테스트 작성

### 커밋 단위
- 기능 단위로 작은 커밋
- 각 커밋은 테스트 통과 상태

---

## 4단계: 리뷰 (Review) 👀

**목표**: 코드 품질 검증

### 셀프 리뷰 체크리스트
- [ ] 요구사항 충족
- [ ] 코딩 컨벤션 준수
- [ ] 테스트 커버리지 충분
- [ ] 보안 취약점 없음
- [ ] 성능 이슈 없음

### 리뷰 요청 시
- PR 설명 상세히 작성
- 변경 이유 설명
- 테스트 방법 안내

---

## 5단계: 테스트 (Testing) 🧪

**목표**: 품질 보증

### 테스트 종류
| 종류 | 목표 | 도구 |
|------|------|------|
| 단위 테스트 | 개별 함수/클래스 | Jest |
| Property 테스트 | 속성 기반 검증 | fast-check |
| 통합 테스트 | 모듈 간 연동 | Jest + DB |
| E2E 테스트 | 전체 플로우 | Supertest |

### 체크리스트
- [ ] 모든 테스트 통과
- [ ] 커버리지 80% 이상
- [ ] 엣지 케이스 테스트
- [ ] 에러 케이스 테스트

---

## 6단계: 문서 (Documentation) 📝

**목표**: 지식 공유 및 유지보수성

### 체크리스트
- [ ] API 문서 업데이트
- [ ] README 업데이트
- [ ] 코드 주석 (복잡한 로직)
- [ ] 변경 이력 기록

### 문서 종류
- API 문서 (Swagger/OpenAPI)
- 아키텍처 문서
- 사용 가이드

---

## 7단계: 배포 (Deployment) 🚀

**목표**: 안전한 릴리스

### 체크리스트
- [ ] 모든 테스트 통과
- [ ] 코드 리뷰 승인
- [ ] 마이그레이션 준비
- [ ] 롤백 계획 수립

### 배포 후
- [ ] 모니터링 확인
- [ ] 에러 로그 확인
- [ ] 성능 지표 확인

---

## 에이전트 역할 정의

### 🔍 Code Explorer
**역할**: 코드베이스 탐색 및 분석
- 관련 파일 찾기
- 의존성 분석
- 패턴 식별

### 📐 Code Architect
**역할**: 설계 및 구조 결정
- 아키텍처 제안
- 인터페이스 설계
- 트레이드오프 분석

### 👀 Code Reviewer
**역할**: 품질 검증
- 코드 리뷰
- 개선점 제안
- 베스트 프랙티스 확인

---

## WPAIS 프로젝트 적용

### 새 기능 시작 시
1. Spec 폴더 생성: `.kiro/specs/<feature-name>/`
2. requirements.md 작성
3. design.md 작성
4. tasks.md 생성
5. Task 순서대로 구현

### 진행 상황 추적
- Task 완료 → GitHub 커밋
- Checkpoint → Notion 업데이트
