# PR 리뷰 툴킷

PR 리뷰 요청 시 6가지 전문 관점에서 분석합니다.

## 6가지 리뷰 관점

### 1. 🏗️ 아키텍처 리뷰어
**관점**: 시스템 설계 및 구조

체크포인트:
- [ ] 모듈 간 의존성이 적절한가?
- [ ] 단일 책임 원칙(SRP) 준수
- [ ] 레이어 분리가 명확한가? (Controller → Service → Repository)
- [ ] 순환 의존성이 없는가?
- [ ] 확장성을 고려한 설계인가?

질문:
- 이 변경이 기존 아키텍처와 일관성이 있는가?
- 향후 요구사항 변경에 유연하게 대응 가능한가?

### 2. ⚡ 성능 리뷰어
**관점**: 실행 효율성 및 리소스 사용

체크포인트:
- [ ] N+1 쿼리 문제 없는가?
- [ ] 불필요한 데이터 로딩 없는가?
- [ ] 적절한 인덱스 사용
- [ ] 메모리 누수 가능성
- [ ] 비동기 처리 적절성
- [ ] 캐싱 고려 여부

질문:
- 대용량 데이터에서도 성능이 유지되는가?
- 병목 지점이 될 수 있는 부분은?

### 3. 🔒 보안 리뷰어
**관점**: 취약점 및 데이터 보호

체크포인트:
- [ ] 입력 검증 완전성
- [ ] SQL 인젝션 방지
- [ ] XSS 방지
- [ ] 인증/인가 확인
- [ ] 민감 데이터 노출 방지
- [ ] 에러 메시지 정보 누출

질문:
- 악의적 입력에 대한 방어가 되어 있는가?
- 권한 없는 접근이 가능한 경로가 있는가?

### 4. 🧪 테스트 리뷰어
**관점**: 테스트 품질 및 커버리지

체크포인트:
- [ ] 단위 테스트 존재
- [ ] 엣지 케이스 테스트
- [ ] 에러 케이스 테스트
- [ ] 테스트 가독성
- [ ] 모킹 적절성
- [ ] 테스트 독립성

질문:
- 테스트가 실제 버그를 잡을 수 있는가?
- 테스트 유지보수가 용이한가?

### 5. 📖 가독성 리뷰어
**관점**: 코드 이해도 및 유지보수성

체크포인트:
- [ ] 네이밍이 명확한가?
- [ ] 함수/메서드 길이 적절
- [ ] 주석이 필요한 곳에 있는가?
- [ ] 복잡한 로직 설명
- [ ] 일관된 코딩 스타일
- [ ] 매직 넘버/스트링 제거

질문:
- 6개월 후에도 이 코드를 이해할 수 있는가?
- 새 팀원이 쉽게 이해할 수 있는가?

### 6. 🎯 비즈니스 로직 리뷰어
**관점**: 요구사항 충족 및 도메인 정확성

체크포인트:
- [ ] 요구사항 완전히 구현
- [ ] 엣지 케이스 처리
- [ ] 비즈니스 규칙 정확성
- [ ] 도메인 용어 일관성
- [ ] 예외 상황 처리

질문:
- 모든 사용자 시나리오가 커버되는가?
- 비즈니스 규칙이 정확히 반영되었는가?
