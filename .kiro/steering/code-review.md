# 코드 리뷰 가이드

코드 리뷰 요청 시 수동으로 적용하는 가이드라인입니다.

## 신뢰도 기반 리뷰 시스템

### 점수 기준
| 점수 | 판정 | 액션 |
|------|------|------|
| 80-100 | ✅ 승인 권장 | 바로 머지 가능 |
| 60-79 | ⚠️ 조건부 승인 | 마이너 이슈 수정 후 머지 |
| 40-59 | 🔄 수정 요청 | 주요 이슈 해결 필요 |
| 0-39 | ❌ 재작성 권장 | 근본적 문제 존재 |

### 점수 산정 요소
- 기능 정확성 (30점)
- 코드 품질 (25점)
- 테스트 커버리지 (20점)
- 보안 (15점)
- 성능 (10점)

## 리뷰 체크리스트

### 🎯 기능 (Functionality)
- [ ] 요구사항 충족 여부
- [ ] 엣지 케이스 처리
- [ ] 에러 핸들링 완전성
- [ ] 비즈니스 로직 정확성

### 📝 코드 품질 (Code Quality)
- [ ] 네이밍 컨벤션 준수
- [ ] 단일 책임 원칙 (SRP)
- [ ] 중복 코드 제거 (DRY)
- [ ] 적절한 추상화 수준
- [ ] 주석/문서화 적절성

### 🧪 테스트 (Testing)
- [ ] 단위 테스트 존재
- [ ] 테스트 커버리지 80% 이상
- [ ] 엣지 케이스 테스트
- [ ] 테스트 가독성

### 🔒 보안 (Security)
- [ ] 입력 검증
- [ ] 인증/인가 확인
- [ ] 민감 데이터 처리
- [ ] SQL 인젝션 방지

### ⚡ 성능 (Performance)
- [ ] N+1 쿼리 문제
- [ ] 불필요한 연산
- [ ] 메모리 누수 가능성
- [ ] 적절한 인덱싱

## 리뷰 코멘트 형식

### 필수 수정 (Blocking)
```
🚨 [MUST] 설명
이유: ...
제안: ...
```

### 권장 수정 (Non-blocking)
```
💡 [SHOULD] 설명
이유: ...
제안: ...
```

### 질문/논의
```
❓ [QUESTION] 질문 내용
```

### 칭찬
```
👍 [NICE] 좋은 점
```

## WPAIS 프로젝트 특화 규칙

### 스케줄링 엔진 리뷰 시
- 일정 계산 로직 정확성
- 의존성 처리 검증
- 리소스 할당 최적화

### 워크플로우 엔진 리뷰 시
- 상태 전이 유효성
- 이벤트 핸들링 완전성
- 동시성 처리

### 공통
- TypeORM 엔티티 관계 설정
- NestJS 모듈 구조
- DTO 검증 데코레이터
