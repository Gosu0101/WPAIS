# PR 리뷰 에이전트 가이드

PR 리뷰 요청 시 수동으로 적용하는 전문 리뷰 가이드입니다.

## 신뢰도 기반 리뷰 시스템

### 신뢰도 점수 (0-100)
- **91-100**: 확실함 - 반드시 수정 필요
- **80-90**: 높은 확신 - 수정 권장
- **50-79**: 중간 확신 - 검토 필요
- **25-49**: 낮은 확신 - 참고용
- **0-24**: 거짓 양성 가능성 높음

**기본 임계값: 80** (80 이상만 보고)

### 거짓 양성 필터링
다음은 보고하지 않음:
- PR에서 도입되지 않은 기존 이슈
- 버그처럼 보이지만 아닌 코드
- 사소한 지적
- 린터가 잡을 이슈
- lint ignore 주석이 있는 이슈

## 병렬 에이전트 리뷰 (4개 에이전트)

### 에이전트 #1, #2: 가이드라인 준수 검사
- steering 파일 규칙 준수 확인
- 프로젝트 컨벤션 검증
- 중복 검사로 신뢰도 향상

### 에이전트 #3: 버그 탐지
- 변경된 코드의 명백한 버그만 탐지
- 기존 이슈는 무시
- 새로 도입된 문제에 집중

### 에이전트 #4: 히스토리 분석
- git blame으로 컨텍스트 파악
- 변경 이력 기반 이슈 탐지
- 패턴 위반 감지

## 사용 가능한 리뷰 관점

### 1. 테스트 분석 (Test Analyzer)

**초점**: 테스트 커버리지 품질과 완전성

**분석 항목**:
- 행동 기반 커버리지 vs 라인 커버리지
- 크리티컬 갭 식별
- 테스트 품질과 회복력
- 엣지 케이스와 에러 조건

**점수 기준** (1-10):
- 9-10: 데이터 손실, 보안 이슈, 시스템 장애 가능성
- 7-8: 사용자 대면 에러 가능성
- 5-6: 혼란이나 마이너 이슈 가능성
- 3-4: 완전성을 위한 추가
- 1-2: 선택적 개선

**트리거 문구**:
```
"테스트가 충분한지 확인해줘"
"테스트 커버리지 리뷰해줘"
"크리티컬한 테스트 갭이 있나?"
```

---

### 2. Silent Failure 헌터

**초점**: 에러 핸들링과 조용한 실패

**분석 항목**:
- catch 블록의 조용한 실패
- 부적절한 에러 핸들링
- 부적절한 fallback 동작
- 누락된 에러 로깅

**심각도 분류**:
- CRITICAL: 조용한 실패, 광범위한 catch
- HIGH: 부적절한 에러 메시지, 정당화되지 않은 fallback
- MEDIUM: 누락된 컨텍스트, 더 구체적일 수 있음

**트리거 문구**:
```
"에러 핸들링 리뷰해줘"
"조용한 실패 확인해줘"
"catch 블록 분석해줘"
```

---

### 3. 타입 설계 분석

**초점**: 타입 설계 품질과 불변성

**평가 항목** (각 1-10점):
- 타입 캡슐화
- 불변성 표현
- 타입 유용성
- 불변성 강제

**트리거 문구**:
```
"타입 설계 리뷰해줘"
"이 타입이 강한 불변성을 가지고 있나?"
"데이터 모델 분석해줘"
```

---

### 4. 코드 리뷰어

**초점**: 프로젝트 가이드라인 준수

**분석 항목**:
- 스타일 위반
- 버그 감지
- 코드 품질 이슈

**신뢰도 점수** (0-100):
- 91-100: 크리티컬 (즉시 수정)
- 80-90: 중요 (수정 권장)
- 80 미만: 보고하지 않음

**트리거 문구**:
```
"최근 변경사항 리뷰해줘"
"커밋 전에 확인해줘"
"코드 품질 체크해줘"
```

---

### 5. 코드 단순화

**초점**: 코드 단순화와 리팩토링

**분석 항목**:
- 코드 명확성과 가독성
- 불필요한 복잡성과 중첩
- 중복 코드와 추상화
- 프로젝트 표준과의 일관성

**트리거 문구**:
```
"이 코드 단순화해줘"
"더 명확하게 만들어줘"
"리팩토링 제안해줘"
```

```
1. 테스트 커버리지 확인
2. 조용한 실패 체크
3. 타입 설계 검토
4. 코드 품질 리뷰
5. 단순화 가능 여부
```

## 종합 PR 리뷰 실행

### 전체 리뷰 요청
```
PR 리뷰 실행:
1. 4개 에이전트 병렬 실행
2. 각 이슈에 신뢰도 점수 부여
3. 80 이상 이슈만 보고
4. 코드 링크 포함

리뷰 대상: [PR 번호 또는 브랜치]
```

### 리뷰 결과 형식
```markdown
## 코드 리뷰 결과

발견된 이슈 3개:

1. [신뢰도: 95] OAuth 콜백 에러 핸들링 누락
   - 위치: src/auth.ts#L67-L72
   - 가이드라인: "항상 OAuth 에러 처리"

2. [신뢰도: 88] 메모리 누수: OAuth 상태 정리 안됨
   - 위치: src/auth.ts#L88-L95
   - 원인: finally 블록에서 cleanup 누락

3. [신뢰도: 82] 네이밍 패턴 불일치
   - 위치: src/utils.ts#L23-L28
   - 가이드라인: "함수는 camelCase 사용"
```
